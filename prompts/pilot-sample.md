# Pilot Sample

Use this after the graph exists and the branch requires calibration before scale-out.

```text
Run the PILOT_SAMPLE node only.

Requirements:
1. Produce one representative unit of work, not a batch.
2. Explicitly state which standards, packet sections, or domain-profile rules governed the work.
3. Show what this sample is meant to prove about the branch.
4. Do not assume that success on one sample automatically means the branch is ready to scale.
5. At the end, report:
   - what the sample demonstrates
   - what is still uncertain
   - what review nodes should evaluate it next
   - whether scale-out is still blocked
```

## Typical use cases
- one chapter for a book branch
- one feature slice for a coding branch with strong judgment surfaces
- one scene for a narrative branch
- one representative design slice for a UX-heavy branch
