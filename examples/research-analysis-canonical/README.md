# Research Analysis Canonical Example

This example pack shows how **AI Task Graph** applies to a current-reality analytical question.

## Scenario

A user asks a question such as:
- can I go to the moon?
- what realistic options exist today for a decision?
- what route is best under current external constraints?

In this class of task, the main risk is not writing quality collapse. It is:
- wrong framing
- stale or missing facts
- solving a nearby question instead of the real one

## Why this example matters

This example highlights a different graph style than writing or coding:
- early question framing
- route-family or question-cluster decomposition
- `DEEP_RESEARCH_AUDIT` before synthesis
- graph refresh if research changes what the real question is

- [`task.md`](task.md) — the incoming analytical task and risks

## What this example proves

A naïve workflow often jumps straight to answer synthesis.

This framework instead forces:
1. precise problem framing
2. decomposition into meaningful investigative routes
3. authoritative research before synthesis
4. explicit recognition that research may change the framing itself

That is the key value of the graph in this domain.
